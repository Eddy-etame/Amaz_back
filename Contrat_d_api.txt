### Contrat d’API – Version 1.2 (PoW sur tous les endpoints externes)

**Conventions générales (mises à jour)**
- PoW obligatoire sur **toutes les routes externes** (register, login, produits, commandes, messages, AI, bot, etc.).
- Header requis sur chaque requête : `X-PoW-Proof` (string calculé par le client).
- Le Gateway applique le middlewarePreuveDeTravail en premier sur toutes les routes (comme dans le code du TP Auth).
- Erreurs toujours génériques et user-friendly en français (« Identifiants incorrects », « Preuve invalide », « Accès refusé », « Erreur interne »).
- Authentification : Tokens HMAC-SHA256 maison (Bearer).
- Internal calls (Gateway → microservices) : mTLS + headers X-User-Id / X-User-Role (pas de PoW, car confiance mutuelle).

**Endpoints Authentification (Gateway)**

- **POST /api/v1/auth/register**  
  Body : `{ "email": string, "password": string (min 8), "username": string (optional) }`  
  Headers : `X-PoW-Proof` (obligatoire)  
  Response 201 : `{ "userId": UUID, "email": string, "role": string }`  
  Erreurs : 400 « Champs manquants », 403 « Preuve invalide », 409 « Conflit données », 500 « Erreur interne ».

- **POST /api/v1/auth/login**  
  Body : `{ "email": string, "password": string }`  
  Headers : `X-PoW-Proof` (obligatoire)  
  Response 200 : `{ "token": string (HMAC), "userId": UUID, "role": string }`  
  Erreurs : 401 « Identifiants incorrects », 403 « Preuve invalide ».

- **GET /api/v1/auth/me**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)  
  Response 200 : `{ "userId": UUID, "email": string, "username": string, "role": string }`

**Endpoints Utilisateurs**

- **GET /api/v1/users/:userId**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)  
  Response 200 : `{ ... }`

- **PUT /api/v1/users/:userId/role**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

- **DELETE /api/v1/users/:userId**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

**Endpoints Produits**

- **GET /api/v1/produits** (recherche paginée)  
  Headers : `X-PoW-Proof` (obligatoire) – même pour les routes publiques

- **GET /api/v1/produits/:produitId**  
  Headers : `X-PoW-Proof` (obligatoire)

- **PUT /api/v1/produits/:produitId**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

- **DELETE /api/v1/produits/:produitId**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

**Endpoints Commandes**

- **GET /api/v1/commandes**

**Endpoints Extras (Messagerie, AI, Bot)**

- **POST /api/v1/messages**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

- **GET /api/v1/messages/:produitId**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

- **POST /api/v1/ai/recommendations**  
  Headers : `Authorization: Bearer <token>` + `X-PoW-Proof` (obligatoire)

- **POST /api/v1/bot/auth**  
  Headers : `X-PoW-Proof` (obligatoire)

**Endpoints Internes (Gateway → Microservices – mTLS uniquement)**  
Pas de PoW (confiance interne).  
Exemples :  
- Users Service : POST /internal/users  
- Products Service : GET /internal/produits/check-stock  
- Orders Service : POST /internal/commandes
